Git Basics

January 8, 2019 2 minutes read
Development • git
git • development

Initialize repo from exiting code

git init

To check status of files

Before commit you can check status of files, like tracked/untracked

git status

.gitignore

Add entries of files in .gitignore you want to exclude from commit.

Create a file

touch .gitignore

Mention whatever you want to exclude like

.data
.project
*.pyc
.ipynb_checkpoints
*/.ipynb_checkpoints/*

Add files to staging area

To add all files

git add -A

To add individually

git add <file>
git add .gitignore

git status

Remove files from staging area

git reset <file>

To reset all files

git reset

First commit

git add -A
git commit -m "Initial version"
git status
git log

Log shows hash code, author, date of commit

Cloning a remote repo

git clone <repo_url> <where_to_clone>

Viewing info about remote repo

git remote -v

To see all the branches

git branch -a

Pushing changes (commit changes)

git diff
git status
git add -A
git commit -m "modification history"

Then Push

git pull origin master
git push origin master
  • origin - is the name of remote repo
  • master - branch to which we want to push

COMMON WORKFLOW

Create a branch for desired feature

git branch <branch_name>

Switch to the new branch

git checkout <branch_name>

To see list of branch

git branch

Make changes and commit locally

Push branch to remote repo

git push -u origin <branch_name>
git branch -a
  • -u to associate the new branch to the remote repo So that next time we can ommit (origin and branch_name) and just do git pull/push from this branch

Merge branch with master

git checkout master
git pull origin master

See what branches are merged

git branch --merged

Push

git merge new_branch_name
git push origin master

Deleting the branch

git branch --merged
git branch -d new_branch_name
git branch -a
git push origin --delete new_branch_name
